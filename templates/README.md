# FIAR-Saúde — Submissão de Projetos e Templates de Documentação

Este repositório disponibiliza os **templates de documentação utilizados no processo de auditoria do framework FIAR-Saúde**.

Os documentos devem ser preenchidos pelas equipes dos projetos participantes para registrar evidências relacionadas às dimensões de **Inteligência Artificial Responsável**.

**Importante**

Ao clicar nos links abaixo, será criada **uma cópia do documento no seu Google Drive**, que poderá ser editada pela equipe do projeto.

Após o preenchimento, compartilhe o link do documento com a equipe de auditoria.

---


## 1. Documentação Inicial do Projeto

Antes do início da auditoria, cada projeto deve preencher o **Project Dossier**, que registra:

- informações gerais do projeto
- escopo do modelo
- equipe responsável
- evidências iniciais relacionadas às dimensões de IA responsável
- links para artefatos do projeto

Criar uma cópia do template:

[FIAR Documentação Inicial do Projeto](https://docs.google.com/document/d/1413BabalmWr3lH9xqPT37aNQ3yF7m-PL-qYjOsisA7I/copy)

Este documento deve ser preenchido **antes do primeiro ciclo de auditoria**.

---

## 2. Relatórios por Dimensão

Após a análise inicial, a equipe de auditoria poderá solicitar relatórios adicionais para cada dimensão do framework.

Esses relatórios documentam **evidências mais detalhadas sobre o sistema de IA**.

---

### Justiça (Fairness)

Documenta a avaliação de possíveis **disparidades entre grupos relevantes**.

Exemplos de conteúdo:

- identificação de subgrupos
- métricas de fairness
- análise de disparidades
- interpretação dos resultados

Criar uma cópia do template: [Fairness Audit Report](https://docs.google.com/document/d/SEU_LINK/copy)
